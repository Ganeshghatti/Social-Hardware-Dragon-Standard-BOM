⚙️ 1. Project Overview
The Robotic Arm Manufacturing Dashboard is an internal web platform to manage the production of a 7 Degrees of Freedom (7DOF) robotic arm.
It centralizes manufacturing data — from parts and suppliers to assembly BOMs and production work orders — into a single, intuitive dashboard.
This is a single-user (admin-only) system for now, designed to scale later into a multi-role manufacturing suite.
________________


🧭 2. Goals & Scope
🎯 Primary Goals
* Build a secure admin dashboard using Next.js + MongoDB.

* Provide clean management interfaces for:

   * Parts and inventory

   * Assemblies and BOMs

   * Work orders and QC processes

   * Suppliers and purchase orders

      * Generate reports (BOM cost, inventory value, work order summaries).

🚫 Out of Scope
         * No file uploads or CAD previews.

         * No real-time socket updates.

         * No external supplier API integrations (future).

🖥️ 4. Pages & Features
🔐 4.1 Login Page
Route: /admin/login
                                    * Fields: email, password

                                    * Button: “Login” (orange)

                                    * Logo on top center (/public/logo.png)

                                    * Auth success → redirect /admin/dashboard

                                    * Invalid creds → show toast: “Invalid email or password”

Backend API:
 POST /api/auth/login
 Returns JWT (stored as HTTP-only cookie).
________________


📊 4.2 Dashboard Overview
Route: /admin/dashboard
 Purpose: A bird’s-eye view of factory data.
UI:
 Grid of summary cards (Tailwind grid-cols-3 md:grid-cols-4):
                                       1. Total Parts in Stock

                                       2. Active Work Orders

                                       3. Pending POs

                                       4. QC Issues This Month

                                       5. BOM Versions

                                       6. Low Stock Alerts

Section 2: “Recent Activity” — show last 10 admin actions from audit_logs.
________________


⚙️ 4.3 Parts Management
Route: /admin/parts
UI Components:
                                          * Table listing all parts

                                          * “Add Part” button opens modal

                                          * Editable fields inline or modal

Table Columns:
 Item Code | Name | Unit | Cost | Stock | Supplier | Reorder Point | Lead Time
Actions:
                                             * Add / Edit / Delete parts

                                             * Search bar (fuzzy search)

Backend APIs:
                                                * GET /api/parts

                                                * POST /api/parts

                                                * PATCH /api/parts/:id

                                                * DELETE /api/parts/:id

🧮 5. MongoDB Schema Overview
Collection
	Fields
	users
	{ email, password_hash, role, last_login }
	parts
	{ item_code, name, cost, stock_qty, reorder_point, supplier_id }
	boms
	{ product_id, type, version, items:[{ part_id, qty }] }
	workorders
	{ product_id, qty, status, schedule_start, schedule_end }
	suppliers
	{ name, contact_person, lead_time_days, rating }
	purchase_orders
	{ supplier_id, items:[{ part_id, qty, cost }], status }
	inventory_transactions
	{ part_id, qty_change, reason, date }
	qc_records
	{ part_id, work_order_id, inspector, pass_fail, remarks }
	audit_logs
	{ user_id, action, collection, document_id, timestamp }

🔒 7. Authentication & Security
                                                                                                   * Passwords hashed (bcrypt).

                                                                                                   * JWT stored in HTTP-only cookie.

                                                                                                   * /admin/* routes protected by middleware.

                                                                                                   * CORS restricted to your domain.

                                                                                                   * Helmet for HTTP headers.

                                                                                                   * Input validation via Zod.
